# File Cleanup & Validation Protocol

## Purpose

Ensure every file in the OSE project is **actively advancing the mission**. Dead files create entropy.

## Validation Criteria

For each file, ask:

1. **Is it referenced?** Does something else link to or use this file?
2. **Is it current?** Has it been updated in the last 30 days?
3. **Is it actionable?** Does it contain instructions someone can execute?
4. **Is it unique?** Is there another file doing the same thing?
