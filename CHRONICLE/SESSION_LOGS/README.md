## Log Format

Each session log follows a consistent format:

- **Header**: Date, Operator(s), Duration, Focus
- **Accomplishments**: Categorized list of completed work
- **Commits**: Table of git commits (Git era only)
- **Issues Encountered**: Problems and their resolutions
- **Lessons Learned**: Wisdom for future agents
- **Next Steps**: Carried forward to next session

## Purpose

These logs serve as:

1. **Historical Record**: What was built and when
2. **OSE Wiki Documentation**: Ready for public upload
3. **Multi-Agent Coordination**: Track contributions across agents
4. **Sprint Planning**: Identify velocity and blockers
5. **Agent Onboarding**: New agents can understand context
